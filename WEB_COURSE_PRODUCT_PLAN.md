# Agent Course Studio 产品与实施计划

## 一、产品定位

`Agent Course Studio` 是面向 Agent 开发学习的课程实验室。它不是普通文档站，而是把 `lessons/L01-L12` 中已经整理好的讲义、代码、面试题、拓展作业和毕业项目，组织成一个可浏览、可搜索、可演示、可继续接入实验运行和课程问答的网页产品。

核心目标：

- 让学生按章节路径学习 Agent 开发，不需要在文件树里来回找材料。
- 让老师能够用网页授课，快速切换讲义、实操、作业、面试题和资源。
- 让课程仓库从“资料集合”升级为“可展示的 Agent 学习作品”。
- 保留未来扩展空间：代码实验台、课程 RAG 助手、LangGraph/L12 过程可视化和在线评测。

## 二、目标用户与典型场景

### 1. 学生自学

学生打开网页后，先看到完整课程地图，再进入某一章。每章都能看到：

- 学习目标
- 推荐路径
- 讲义与总结
- 实操脚本
- 小测与拓展作业
- 面试题
- 相关资源

### 2. 老师授课

老师可以把网页作为课堂主屏：

- 使用左侧课程地图切换章节。
- 使用章节 Tab 切换“讲义 / 实战 / 面试 / 资源”。
- 使用搜索快速定位某个概念，例如 RAG、MCP、Skill、回滚、Prompt Injection。
- 使用实验区展示本章脚本如何运行。

### 3. 项目展示与面试

毕业项目 L12 可以作为作品核心入口：

- 展示智能客服 Agent 的架构、验收标准和运行命令。
- 让面试官快速看到课程覆盖范围、工程质量和安全意识。
- 作为后续部署到 GitHub Pages / 自有服务器的作品主页。

## 三、功能分层

### Phase 1：静态课程实验室

第一阶段先完成不依赖外部服务的稳定版本：

- 课程首页和 12 章课程路径。
- 每章内容页：概览、讲义、实战、面试题、资源。
- 全局本地搜索。
- 本地检索式课程助手，用课程索引返回引用来源。
- 学习进度本地保存。
- 实验脚本白名单展示与命令复制。
- 响应式布局，支持桌面与移动端。

### Phase 2：受控实验运行

第二阶段接入本地 Python 服务：

- 只允许运行白名单脚本，例如各章 `practice/preclass_run.sh`。
- 默认关闭真实执行，设置 `AGENT_COURSE_STUDIO_ALLOW_RUN=1` 后才允许。
- 执行前展示命令、目录、风险提示。
- 执行后返回日志、退出码、耗时。
- 不允许浏览器提交任意 shell 命令。

### Phase 3：课程 RAG 助手

第三阶段把当前本地检索助手升级为真正 RAG：

- 对 `README.md`、`lecture/`、`materials/`、`resources/` 建立索引。
- 返回答案时必须带章节和文件引用。
- 支持“按章节问”“按主题问”“按面试问”三种模式。
- 对敏感问题、无证据问题明确回答未找到。

### Phase 4：Agent 过程可视化

第四阶段做课程特色能力：

- Function Calling：展示工具 schema、tool call、tool result。
- RAG：展示 query rewrite、召回 chunk、引用来源。
- Memory：展示短期记忆、长期记忆、摘要记忆的变化。
- LangGraph：展示节点流转和条件边。
- Skill：展示触发条件、加载层级、工具执行、安全拦截。
- L12：展示智能客服 Agent 的意图识别、RAG、工单、人工兜底和评测结果。

## 四、技术策略

### 当前落地策略

本阶段采用“静态前端 + Python 内容构建器 + 轻量本地服务”的方案：

- 不引入大型前端依赖，降低安装和部署成本。
- 直接从现有课程目录生成 `course.json`。
- 生成轻量 manifest、同步 boot 数据和按需加载 Markdown 文档，保证首屏速度和静态部署稳定性。
- 前端使用原生 HTML/CSS/JS，便于 GitHub Pages 或任意静态服务器托管。
- 本地服务使用标准库 `http.server`，额外提供受控运行 API。

### 后续可升级策略

当需要用户系统、在线作业、正式 RAG 和多人使用时，再升级为：

- `apps/web`：Next.js / React。
- `apps/api`：FastAPI。
- `content/`：课程 manifest 与索引。
- `runtime/`：白名单实验运行器。
- `evals/`：课程问答和练习评测集。

## 五、目录设计

```text
apps/
  agent_course_studio/
    README.md
    build_course_data.py
    server.py
    web/
      index.html
      styles.css
      app.js
      data/course.json
      data/course_boot.js
      data/docs/
```

说明：

- `lessons/` 继续作为唯一课程内容源。
- `apps/agent_course_studio/web/data/` 是生成物，可以随课程更新重新生成。
- `course.json` 提供索引，`course_boot.js` 提供首屏同步数据，`docs/` 提供按需加载的 Markdown 文档。
- 前端不修改课程原始 Markdown，只负责展示和检索。

## 六、页面信息架构

### 1. 总览页

- 课程标题与一句话定位。
- 课程关键指标：12 章、实践脚本、面试资料、毕业项目。
- 课程路径时间线。
- 主题雷达：Prompt、Tool、RAG、Memory、MCP、Skill、Eval、Deploy。
- README 图表：课程路线图、Studio 架构图、能力覆盖图。

### 2. 章节页

每章包含：

- 章节概览
- 学习目标
- 交付物
- 文件地图
- 讲义 / 总结
- 实操脚本
- Agent 开发图谱
- 面试题
- 拓展作业
- 资源专题

### 3. 搜索与课程助手

- 搜索：返回匹配章节、文件和摘要。
- 课程助手：根据本地索引拼出带引用的学习建议，不伪装成真实 LLM。

### 4. 实验区

- 展示本章一键脚本。
- 展示关键实践文件。
- 提供复制命令。
- 本地服务允许时，执行白名单脚本并显示结果。

## 七、安全边界

网页实验运行必须满足：

- 禁止任意命令执行。
- 只允许运行课程生成的白名单脚本。
- 默认关闭真实执行。
- 运行超时受控。
- 不读取或展示 `.env` 内容。
- 不把 API key、代理配置、个人路径写入 `course.json`。

## 八、验收标准

### 功能验收

- 能打开课程实验室首页。
- 能展示 L01-L12 全部章节。
- 每章能看到讲义、实战、面试、资源等核心信息。
- 每章能看到本章 Agent 开发图谱。
- 能搜索课程内容。
- 课程助手能返回带引用的本地检索结果。
- 学习进度能保存在浏览器本地。
- 实验区能展示安全白名单命令。

### 工程验收

- `build_course_data.py` 能重新生成课程索引。
- `build_course_data.py` 能重新生成 `course.json`、`course_boot.js` 和 `data/docs/`。
- `server.py` 能启动本地服务。
- 生成数据不包含 `.env`、真实 API key、个人绝对路径。
- HTML/CSS/JS 没有明显布局错误。
- 在桌面和移动宽度下都可用。

### 文档验收

- 根 README 有课程实验室入口。
- App README 有运行、构建、安全说明。
- 产品计划记录完整，便于后续迭代。

6. 编写轻量本地服务。
7. 更新 README。
8. 启动服务验证。
9. 做安全扫描与最终审计。
